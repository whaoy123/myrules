# myrules

本仓库用于集中存放项目写作和文档整理过程中沉淀下来的通用规则，便于在不同工程之间复用。

当前包含以下规则文件：

| 文件名 | 用途 |
|---|---|
| `chinese-writing-rules.md` | 约束中文技术文档中的常见非标准用词，统一更自然、更正式的表达方式 |
| `word-conversion-rules.md` | 约束 Markdown 方案文档在后续转 Word 交付时的写法，减少排版返工 |

使用方式如下：

1. 在项目文档编写阶段，先按对应规则约束正文写法。
2. 若规则只适用于某一类文档，应在项目仓库中按需引用，而不是机械套用到所有文件。
3. 当项目中出现可复用的新规范时，可继续补充到本仓库并保持规则粒度清晰。
